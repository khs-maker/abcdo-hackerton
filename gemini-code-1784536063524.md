# 2026 ABC 포용적 AI 해커톤 최종 기획안

## 1. 최종 프로젝트명
* **AURA Layer** (Adaptive User Reality-to-Action Layer)[cite: 2]
* **서브타이틀**: 현실의 안내문을 개인별 공공서비스 행동 화면으로 바꾸는 포용형 AI 인터페이스 계층[cite: 2]

## 2. 한 줄 설명
사용자가 고지서·공공 안내문을 촬영하거나 원하는 일을 음성으로 말하면, AI가 현실 상황과 사용자의 현재 어려움을 이해하고 기존 공공서비스 위에 그 사람이 실제로 사용할 수 있는 맞춤형 행동 인터페이스를 생성합니다[cite: 2].

## 3. 주제 피봇(Pivot) 배경 및 당위성
기존의 침수 대응 AI 아이디어는 단발성 재난 상황 외에는 사용 빈도가 낮고, 단순 정보 제공형 챗봇에 갇히는 한계가 있었습니다[cite: 2]. 
AURA Layer는 이 약점을 보완하여 **모든 공공서비스로 확장 가능한 '기반 접근성 기술'**로 전환하였습니다[cite: 2]. 정보 제공에서 끝내지 않고 사용자가 실제 민원이나 복지 신청 과업을 최종 완료하도록 단계별 실행을 안전하게 지원합니다[cite: 2].

## 4. 해결하려는 문제
정부24와 복지로 등 기존 공공 플랫폼에는 이미 민원 검색 및 맞춤형 혜택 안내 기능이 존재하므로, 단순히 "서비스를 찾아준다"는 접근만으로는 차별성이 부족합니다[cite: 2]. 
남아 있는 핵심 문제는 **'적절한 서비스를 찾았더라도 복잡한 행정용어, 관료제적 메뉴 구조, 수많은 입력 항목 때문에 디지털 취약계층(고령자, 외국인 등)이 중도 포기한다'**는 점입니다[cite: 2]. AURA Layer는 새로운 서비스를 중복 구축하는 대신, 기존 서비스와 사용자 사이에 '포용형 AI 인터페이스 레이어'를 추가하여 이 장벽을 허뭅니다[cite: 2].

## 5. 핵심 사용자 및 타깃 도움 모드
사용자에게 장애나 질환을 직접 묻지 않고, 현재 상황에서 필요한 도움을 선택하게 하여 낙인 효과를 최소화합니다[cite: 2].
* **쉽고 짧게 알려주세요**: 행정용어 해설 및 인지 부담 완화[cite: 2]
* **한 번에 하나씩 보여주세요**: 복잡한 절차의 마이크로 과업 분할[cite: 2]
* **큰 글씨로 보여주세요**: 저시력 사용자를 위한 고대비 UI 지원[cite: 2]
* **음성으로 읽어주세요**: Web Speech API 기반 문장 단위 음성 안내[cite: 2]
* **한국어가 어려워요**: 외국인 사용자를 위한 행동 중심 다국어 변환[cite: 2]

## 6. 대표 MVP 시나리오 및 UI 컴파일러 통제 전략
프런트엔드 깨짐이나 대형 언어 모델(LLM)의 예측 불가능한 예외 출력을 방지하기 위해, AI는 정밀 구조화된 JSON 데이터만 생성하며 프런트엔드는 미리 검증된 **5대 정적 UI 블록**만 조합하여 렌더링합니다.

* **핵심 컴포넌트 5종 규격**:
  1. `QuestionCard`: 한 화면에 질문 1개, 선택지 최대 2~3개 제한 (예/아니오/모름)[cite: 2]
  2. `ChecklistCard`: 준비물 및 필요 서류 목록을 체크박스 형태로 직관적 제공[cite: 2]
  3. `LinkCard`: 출처가 검증된 공식 기관 신청 페이지 연결[cite: 2]
  4. `ActionCard`: 관공서 전화 걸기 연동 및 가족/보호자 원격 공유 기능[cite: 2]
  5. `AudioPlayerCard`: 음성 안내 재생, 반복 및 속도 조절 컨트롤러[cite: 2]

## 7. 플랫폼 확장성 시나리오 (다문화가정 및 외국인)
동일한 공공 안내문을 외국인 사용자가 촬영했을 때, 단순 단어 직역을 넘어 국내 행정기관의 역할 설명을 덧붙이고, 해당 관공서에 전화를 걸었을 때 바로 읽을 수 있는 '한국어 발음 및 영문 대조 소통 스크립트'를 생성하여 실질적인 행동 완수를 지원합니다[cite: 2].

## 8. 핵심 기술 구조 (System Architecture)
1. **현실 입력 (Reality Input)**: 안내문 사진(멀티모달) 또는 음성 요청 수렴[cite: 2]
2. **AI 분석 엔진 (Gemini)**: 문서 목적 및 사용자 의도 분석, 구조화 JSON 추출[cite: 2]
3. **공식 정보 검색 레이어**: 정부24, 공공데이터포털 연동을 통한 정보 진위 검증[cite: 2]
4. **규칙 및 안전성 검증**: 자격 단정 방지 및 가짜 정보 필터링[cite: 2]
5. **접근성 계획 엔진**: 사용자의 도움 모드에 따른 정보량 및 컴포넌트 시퀀스 결정[cite: 2]
6. **UI Compiler**: 검증된 5대 정적 UI 블록을 동적으로 매핑 및 조합[cite: 2]
7. **Action Layer**: 공식 웹사이트 연동, 전화(`tel:`), Web Share API 실행[cite: 2]

## 9. 데이터셋 전략 (AURA Public Task Dataset)
공공누리 또는 공공데이터 이용조건을 준수하여 직접 가공한 데이터셋을 대회 핵심 제출물로 포함합니다[cite: 2].
* **구성**: 실제 공공 안내문 및 고지서 샘플 이미지 30개[cite: 2], 대표 행정 과업 시퀀스 20개[cite: 2], 조건별 정답 행동 JSON 스키마 구조체 30세트[cite: 2].

## 10. 기술적 검증 실험 (AI 에이전트 시뮬레이션 및 가산점 전략)
대회 기간 내 피실험자 대면 모집 리스크를 기술로 정면 돌파하기 위해 **'AI 에이전트 가상 페르소나 시뮬레이션'**을 주 검증 지표로 채택합니다.
* **정량적 실험 (AI 샌드박스 테스트, N=100+)**: Gemini API에 디지털 취약계층(75세 고령자, 행정 용어가 서툰 외국인 등)의 페르소나 프롬프트를 주입한 에이전트를 구축합니다. 기존 공공 사이트 이용 환경과 AURA Layer 환경을 대조하여 과업 성공률, 마우스 클릭 수, 인지 오류율 등의 로그 데이터를 대량 수집해 정량적 그래프를 논문에 확보합니다.
* **정성적 실험 (게릴라 지인 테스트, N=3)**: 팀원의 실제 고령 가족 3명을 대상으로 간이 테스트 및 인터뷰를 진행하여 실제 정성적 코멘트(FGI)를 보완합니다[cite: 2].
* **성공 목표 지표**: 과업 성공률 95% 이상, 기존 웹 대비 과업 완료 시간 40% 이상 단축, 클릭 수 50% 이상 감소[cite: 2].

## 11. 안전성 및 신뢰성 정책 (Responsible AI)
1. **자격 최종 확정 금지**: AI가 수혜 자격을 최종 단정 짓지 않고 공식 기준에 따른 가능성만 안내[cite: 2].
2. **데이터 최소화**: 주민등록번호 등 민감한 개인정보는 저장하지 않으며 모크(Mock) 데이터로만 처리[cite: 2].
3. **상시 이탈 보장**: 사용자가 혼란을 느낄 경우 언제든지 순정 공공 웹 화면으로 돌아갈 수 있는 버튼 배치[cite: 2].
