# AURA Layer 기획서 초안

## 프로젝트명
* **국문**: 아우라 레이어 (AURA Layer)[cite: 3]
* **한 줄 설명**: 현실의 공공 안내문을 이해하여 사용자에게 필요한 맞춤형 디지털 UI를 생성하고, 공식 서비스와 다음 행동(Action)을 안전하게 연결하는 포용형 인터페이스[cite: 3].

## 1. 문제 정의
현대 사회에는 정부24, 복지로, 국민건강보험 등 수많은 공공 디지털 서비스가 매우 잘 구축되어 있습니다[cite: 3]. 따라서 문제는 '정보나 서비스의 부족'이 아닙니다[cite: 3]. 
진짜 문제는 **'현실에서 종이 안내문이나 고지서를 받았을 때, 복잡한 행정용어와 까다로운 웹사이트 메뉴 구조 때문에 내가 지금 정확히 무엇을 해야 하는지 판단하기 어렵다'**는 점입니다[cite: 3]. 

특히 고령자, 외국인, 디지털 취약계층은 필요한 서비스를 찾고, 정보를 해석하고, 실제 신청 행동으로 옮기는 전 과정에서 큰 장벽을 마주합니다[cite: 3]. 본 프로젝트는 이를 단순한 접근성 문제가 아닌 **'현실(Reality)과 디지털 서비스 사이의 인지적 단절'**로 정의합니다[cite: 3].

## 2. 핵심 철학 및 비전
* **사용자 중심 전환 (User-Centered)**: 기존 디지털 환경은 사용자가 시스템 구조를 직접 배워야 하는 '서비스 중심'이었습니다[cite: 3]. AURA는 서비스가 사람의 상황과 능력에 맞게 스스로 형태를 바꾸는 '사용자 중심' 인터페이스를 지향합니다[cite: 3].
* **Reality → Context → Action → Official Service**: 현실의 안내문 촬영에서 출발하여(Reality)[cite: 3], 사용자의 디지털 숙련도를 반영해 상황을 이해하고(Context)[cite: 3], 5대 정적 UI 블록으로 쪼개어 행동을 제안한 뒤(Action)[cite: 3], 최종 신청은 안전하게 공식 서비스로 연계합니다(Official Service)[cite: 3].
* **독립적 인터페이스 계층**: AURA는 기존 공공 플랫폼을 대체하거나 경쟁하지 않으며, 사용자가 공식 서비스를 안전하고 쉽게 이용할 수 있도록 돕는 투명한 레이어(Interface Layer) 역할을 수행합니다[cite: 3].

## 3. 시스템 구성 요소
* **사용자 인터페이스 (AURA UI)**: 이미지(멀티모달 촬영) 및 음성 입력을 직관적으로 수용하는 단순화된 화면[cite: 3].
* **AI 분석 모듈 (Gemini API)**: 입력된 멀티모달 데이터를 기반으로 문서의 핵심 목적을 추출하고, 프런트엔드가 제어 가능한 구조화 JSON 데이터로 변환[cite: 3].
* **행동 생성 및 UI 컴파일러**: 분석 결과를 바탕으로 사전에 정의된 안전한 컴포넌트(질문 카드, 서류 체크리스트 등)를 동적 조합하여 단계별 가이드 화면 구성[cite: 3].
* **공식 서비스 연계**: 최종 신청 및 민원 처리는 정부24 등 공공기관 공식 홈페이지 인프라로 연결하여 보안성 및 신뢰성 확보[cite: 3].

## 4. MVP (Minimum Viable Product) 구현 범위
* **포함 범위 (In-Scope)**:
  * 실제 공공 안내문 이미지 업로드 및 멀티모달 기반 목적 분석[cite: 3]
  * 인지 과업 분할 알고리즘 및 5대 핵심 UI 블록 기반 동적 화면 렌더링
  * 정부24 및 복지로 등 공식 안내 페이지 및 민원 연결 인프라 연동[cite: 3]
  * 가상 AI 페르소나 에이전트 기반 접근성 시뮬레이션 검증 샌드박스 구현
* **제외 범위 (Out-of-Scope)**:
  * 관공서 시스템 내부의 개인정보 자동 대리 입력 및 대리 신청 기능[cite: 3]
  * 자체 회원가입 시스템 및 영구 개인정보 DB 구축[cite: 3]

## 5. 기대 효과
* **사용자 측면**: 여러 사이트를 오가며 정보를 종합할 필요 없이, 촬영 한 번으로 나에게 최적화된 마이크로 과업 인터페이스를 제공받아 인지적 부담과 탐색 시간이 극적으로 줄어듭니다[cite: 3].
* **사회적 측면**: 취약계층용 별도 플랫폼을 신설하는 비용을 아끼고, 기존에 구축된 국가 전자정부 인프라의 접근성과 활용 효율성을 극대화하여 진정한 '기술적 포용'을 실현합니다[cite: 3].
* **기술적 측면**: AI를 단순한 텍스트 챗봇이 아닌, 실시간 인터페이스 생성기(UI Compiler) 및 접근성 자동 검증 도구(AI Simulation)로 활용하는 독창적인 기술적 선례를 제시합니다[cite: 3].
